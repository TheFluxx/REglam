# 06 Quality Gates

## Назначение
Определяет блокирующие критерии итоговой приемки `Pass/Fail`.

## Gate Defects
1. `Critical/Open = 0` обязательно.
2. `Major/Open = 0` или есть `Accepted with rationale` и `Expiry` в `docs/04_decisions.md`.
3. Любой `Critical/Open` автоматически переводит итог в `Fail`.

## Gate UX
1. Обязательны два сценария: основной и альтернативный.
2. Для каждого сценария обязательны три прохода:
- happy-path;
- invalid-path;
- recovery-path.
3. Неполное покрытие сценариев делает gate = `FAIL`.

## Gate A11y
1. Клавиатурная проходимость основных интерактивных элементов.
2. Видимый `focus-state` на интерактивных элементах.
3. Логичная иерархия заголовков.
4. Осмысленные `alt` у контентных изображений.

## Gate Performance
1. Нет критических ошибок консоли в основном сценарии.
2. Нет тяжелых неиспользуемых ассетов.
3. Между `v04` и `RC` не зафиксирована деградация первой отрисовки.

## Gate Content
1. Единый тон сообщений и CTA.
2. Нет заглушек типа `lorem`, `todo`, `tbd` в пользовательском интерфейсе.
3. Сообщения об ошибках содержат понятное действие.

## Правило финальной приемки
`Pass` ставится только если все gates имеют `PASS`.  
Иначе итог: `Fail` или `Fail (Conditional)` при формальном принятии риска с `Expiry`.
